Blogdown

The blogdown package is superb. I created this site by installing Hugo via

blogdown::install_hugo()

and then, in a folder for a github repo, running the below.

blogdown::new_site(theme = 'yihui/hugo-xmin')

The xmin theme (GitHub) is simple. Keeping it simple allows me to customise and, maybe, fix any future errors.

Then I can open up a new file for creating a post,

blogdown::new_post(title='Container for blogdown?')

add the content and generate the site.

blogdown::serve_site()

That command will also create a web server process so you can see the website.

Netify

As an aside, I use the Netify service to publish the website. The Blogdown package puts the files for the page in a folder called public. I upload all the files created by Blogdown to a GitHub repository. Netify creates a container with a webserver and imports the files in public. There are more details about blogdown and Netify here.

Containers

Imagine defining an isolated Linux system using a text file. You can run that system on any Linux, MacOS or Windows machine. The same environment, every, darn time.

Sounds useful, huh?

Containers are that solution. There are several programs for running containers (e.g., Docker, Podman, Apptainer). To get started, I suggest using Docker desktop.

Containers are defined using a Dockerfile. Let’s consider a simple example where we extend the Ubuntu bionic docker image and upgrade the packages.

FROM ubuntu:bionic

RUN apt update
RUN apt -f upgrade
